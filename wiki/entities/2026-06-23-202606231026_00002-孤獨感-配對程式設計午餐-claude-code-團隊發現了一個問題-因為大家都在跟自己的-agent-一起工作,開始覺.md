## 原始貼文
孤獨感：配對程式設計午餐
Claude Code 團隊發現了一個問題：因為大家都在跟自己的 agent 一起工作，開始覺得孤獨了。所以他們最近開始了配對程式設計午餐（pair-wise programming lunch）。他們發現每個人使用 Claude Code 和 Cowork 的方式都不一樣，配對工作時能從彼此身上學到很多。Hackathon 也是他們確保團隊互動的方式之一。
Lenny 說這是一個很棒的觀察。過去工程團隊一起寫程式，有人做後端、有人做前端、有人做 iOS，大家互相依賴彼此協作。現在變成十個 Claude 在平行處理各種事情，每個工程師跟自己的 agent 組成一個小小的獨立單位。配對程式設計午餐的概念就像小孩子的「平行遊戲」（parallel play），你在旁邊做自己的事，但光是看別人怎麼做就很有收穫。
Lenny 也好奇在這個新世界裡還有什麼東西失去了。他自己當了十年工程師，最懷念的是那種進入心流的感覺，音樂放上去，整個人沉浸在程式碼裡，看到它編譯成功的那一刻太爽了。Fiona 說她跟團隊裡的工程師也聊過這個，那種花了好幾個小時終於破解一個棘手問題的「啊哈」時刻，確實是很珍貴的。現在大家更多的喜悅來自產品本身，但她也聽到其他工程師說「我以前最享受的就是那些最困難的部分」。大家都在調適中。
Fiona 說她自己每次看團隊成員工作都會學到新東西，因為團隊裡自己的工具鏈變化太快了。至於有沒有人太沉迷於優化工作流程？她說團隊大家都忙著做架構更新或產品體驗，午餐時間的討論很有趣，但沒有人過度優化，因為沒有完美答案，而且有太多事情要做了。

▍ 情境切換的負荷
Lenny 問 Fiona 還有什麼最近改變了、需要重新思考運作方式的。Fiona 說因為 routines 和更多非同步工作，情境切換的負荷正在增加。不管是團隊成員還是使用者，他們都需要想辦法讓這個體驗更好、降低這個負荷。她自己有一次啟動了太多非同步 agent，才意識到這是個問題。以前她會刻意留下專注時間來寫程式，現在因為可以同時跑更多非同步 agent，她發現反而需要刻意留下專注時間來追趕所有非同步工作的進度。
Lenny 說如果你有 20 個 agent 同時在跑，要不斷切換去 check in 和 review，還得記得每邊在做什麼。有趣的是 agent 可以幫你重新進入狀態，「這是你之前在做的」，所以切換的重啟成本其實降低了，因為你不用重新理解整個 codebase 和架構。某種程度上變好了也變差了。Fiona 坦承她還沒破解這個問題。
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
▍ PM 和 Data Science 角色的轉變
所有跟 coding 相鄰的角色都在轉變。PM 的轉變特別大，因為 PM 不再因為等待工程資源而卡住，他們有想法就可以自己動手做。Anthropic 的 PM 甚至幫忙出貨一些功能。但驗證就變得更重要了，因為更多不同職能的人在 check in code，要怎麼確保大家都有信心？
她認為在 coding 之外，設計和 data science 是下一波可以大幅改善體驗的領域。團隊需要持續自動化工作流程中 coding 以外的其他環節。Lenny 分享了他一個做 data science 的朋友的故事：現在大部分工作變成別人用 AI 自己做了不太好的數據分析，然後拿來問 data scientist「你幫我看看對不對？」然後一半的時候是不對的。所以 data scientist 的工作完全變了，不是自己做分析，而是整天在 review AI 做的分析。
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
▍ Dogfooding 的執念
跟 Fiona 共事過的人提到最多的就是她對 dogfooding 的執念。她解釋說，每次你做產品都有一個夢想，一個你想達成的體驗。親自使用產品讓你能夠保持對那個脈搏的敏感度。不管是 Visual Studio、Marketplace、VR、AR 還是 Instagram，她在每個團隊的第一件事都是使用自己的產品。
她分享了在 Marketplace 時把一台舊 MacBook Air 掛上賣，結果馬上就有買家試圖詐騙她，而且是一種她從沒偵測到的新型態詐騙手法。在做 VR 和 AR 時，她的家裡環境不知為何總是能觸發奇怪的地板高度問題，所以她變成了一個很好的重現環境。
她說作為領導者，如果你只看指標和儀表板或簡報，你會跟產品失去觸感。每次她做客戶拜訪都會學到新東西。她舉了 Marketplace 要拓展到拉丁美洲的例子，在智利的表現不如其他區域。她帶了三人小組去智利，帶了一堆 Android 手機，一落地就發現 LTE 連線比美國慢很多，Marketplace 的動態消息根本載不出來。這就是一個巨大的成長阻礙，但如果不親自去體驗根本不會知道。
Lenny 引用了 Jeff Bezos 的話：「如果你有數據也有軼事，相信軼事而不是數據。」Fiona 非常認同。
如果你負責的產品你很難自己使用，那就去見客戶。每次客戶拜訪她都會學到新東西。她最後一個團隊是 VR 團隊，她雖然沒有 check in 任何程式碼因為太擔心搞壞作業系統，但她把 dogfooding 的時間用來驗證體驗品質，成為團隊品質把關的一個重要環節。
她說作為領導者，團隊成員真的會在意你有沒有在使用他們做的產品。每一次她使用產品並給出使用者回饋，團隊都會很感動，覺得領導者真的投入在裡面，而不是只看指標和簡報。任何能讓你跟使用者體驗保持接觸的管道都很重要。
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
▍ 工程的未來：教育與新世代
Fiona 坦言她有一些脆弱的想法要分享。她真正擔心的大問題之一是：我們怎麼培養下一代？因為你和我走上工程這條路的方式跟現在完全不同了。現在從學校畢業，怎麼快轉那些經驗？她仍然認為理解依賴的那個「雙重點擊」能力很重要，也許模型有一天會強到不需要了，但現在還沒到。
她在想也許軟體工程會走向更像 fellowship 或學徒制（apprenticeship）的方向。過去有暑期實習，三個月做小專案，但也許需要重新思考怎麼把那些人生經驗和工程素養壓縮傳授給下一代建造者。
Lenny 說如果你永遠不需要看程式碼，新工程師有什麼動力去真正理解基礎架構和記憶體配置這些基礎知識？Fiona 說也許模型會強到不需要了，但她覺得那個「雙重點擊」的地方可能藏著改善產品或系統的機會。
她分享了她前主管的故事，他從打孔卡（punch cards）時代開始做軟體工程，現在一直傳訊息給她展示他用 Claude Code 做的東西。從打孔卡到 Claude Code，這種完整的職涯弧線真的很壯觀。也許重要的東西會隨時間改變，我們要持續辨別什麼仍然重要、什麼重要性已經改變，然後在那些真正重要的東西上建立能力。
Fiona 也提到模型進步的速度是指數級的。她記得大概是 Sonnet 3.5 或 3.6 的時候，模型還會犯一些錯誤，有些抗拒 AI 工具的工程師會說「看吧，它就是不行。」但他們沒有理解改進的速度有多快。她自己也在學習，也許之前嘗試自動化但 Claude 還不夠好的東西，在下一個模型版本就夠好了。所以值得定期重新嘗試那些曾經失敗的事情，因為那個能力可能已經到了。Lenny 補充說這在 podcast 上經常出現的觀點：去做那些幾乎快要可以運作的東西，因為一旦模型到了那個水準，你就會遠遠領先所有人。
Lenny 說年輕人的優勢在於他們更容易擁抱新的工作方式，不會被舊的方式卡住。像 Fiona 這樣的資深工程師能適應並擁抱這些改變反而更難得。

▍ 團隊文化：讓 Fiona 睡不著的事
最讓她睡不著的是團隊文化。她很重視「one team」的心態，文化不是貼在牆上的海報，它是活的、會呼吸的，它展現在我們怎麼對待彼此、怎麼互相支持。她希望團隊有多元觀點，能有健康、開放、誠實的辯論。她所謂的 one team 心態是「當你快到終點線的時候回頭看，有沒有團隊的人需要你幫忙，因為我們是一起抵達終點的。」
在快速成長中維持文化是最難的人際面向，不像產品或工程挑戰可以靠儀表板或假說處理。當文化開始偏移，你有沒有及時察覺？有沒有跟團隊一起討論怎麼讓文化往對的方向發展？
Lenny 分享了在 Airbnb 的經驗：創辦人對文化的執念是關鍵，每次全體會議都在提醒文化和價值觀的重要性。他也提到 Sheryl Sandberg 來做 fireside chat 時說：「這其實是你最想要的問題，因為這代表你在成長、做得很好。什麼都不變的情況代表你做得很差，那才是更糟的處境。」
Fiona 特別強調她會要求團隊裡的每個人，特別是管理者，不只分享進展順利的事，也要坦誠什麼沒有進展順利。因為只有當你能談論什麼沒做好，你才能真正去解決它。她的噩夢是有人在管理職位上被問「一切還好嗎？」回答「一切都很好」，然後她就知道「完蛋了，我做得不夠好。」那個坐在咖啡杯前面、房間著火的貓咪迷因就是她的噩夢。所以她在每個新管理者加入時都會說：讓我們永遠保持開放的對話，這樣我們才能一起解決問題。
Lenny 說他能想像很多人都在掙扎於此。當你看到周圍的人看起來都做得超好，至少表面上是這樣，很難坦承說「其實我做得不好，我在某些方面落後了。」因為每個人在社群表面上看起來都在殺四方。Fiona 也承認說，考慮到 Anthropic 這種史無前例的成長軌跡，文化的挑戰是很自然的。即使在 Airbnb 那種增長速度下，文化維護都已經是持續的議題了，而 Anthropic 的增長速度又是另一個量級。

▍ JIT 月度規劃：從六個月路線圖到即時規劃
Fiona 分享了她對流程的一個重要態度：明確允許砍掉不再有用的流程。她建議每個人挑一個你害怕做、很費力、或太手動的流程，先問它是不是還有用。
她自己剛加入 Claude Code 時帶進來一個六個月路線圖的做法，雖然已經刻意做得很輕量。這個練習在啟動對話和確保對齊上很有用，但三個月後她發現大家根本沒有再回去看，因為變化太快了。這讓她學到了一課。
現在她稱之為 JIT planning，即時規劃（just-in-time planning），以一個月為週期。其實連文件都不寫，就是用一個簡單的試算表列出這個月認為最重要的優先事項。甚至她在想每週還要做一個快速 check in，「對，這些還是本月優先事項嗎？好。」但即使這個流程她也在想怎麼進一步自動化，因為她不希望更新試算表變成一種負擔。
每六個月會把整個團隊聚在一起，討論一些主題方向，但實際的規劃和執行是月度的，因為主題也會隨著環境變化而改變。Lenny 說他的 PM 腦袋覺得這個做法很基本，「這個月我們做什麼，每週確認一下，怎麼可能不這樣做？」但他也注意到試算表上的項目應該不多。Fiona 說他們真的很專注，列出最高優先事項，然後每個人根據這些優先事項提出自己的方案，充分發揮 agency。
她提出了一個給所有團隊的建議：明確允許砍掉不再有用的流程。挑一個你害怕做的、很吵的、很貴的、或者很手動的流程，先問它是不是還在服務它原本的目的。永遠要對學習保持開放，永遠要問自己現有的流程是不是仍然有用，因為這個領域變化太快了。
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
▍ 還沒破解的難題
Lenny 提到 Fiona 在演講中列出的三個她還在思考的問題。第一，還需要分開的 iOS 和 Android 組織嗎？Fiona 說深度專業仍然重要，但不需要像以前那麼大的規模，因為人們可以跨領域了。要確保有足夠的 Android 和 iOS 專家，但不需要每個平台各一個大團隊。第二，完全自動化的 review 要推多遠？這跟第三個問題（角色模糊的情況下怎麼確保人人同樣有生產力）一起，仍然是很大的機會空間。她說要持續問自己，哪裡仍然需要專家，同時也要問有沒有辦法把專家的知識也自動化。
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
▍ 工程管理者現在應該期待什麼
Lenny 問工程管理者現在應該對團隊有什麼期待。Fiona 說現在大部分 commit 都有 Claude 輔助，這是一個轉變。工程師需要建立更強的產品敏感度，變成更強的 product engineer。以前那些因為等待跨職能夥伴而卡住的情況也減少了，因為模型能補足你作為工程師可能沒有的額外能力。所以變化是雙向的：工程師變得更有產品思維，同時所有人都在變得更像工程師。角色界線正在全面模糊。

▍ 閃電問答
Lenny 問 Fiona 推薦什麼書。小說方面她推薦兩位作家：Margaret Atwood（她在加拿大長大所以讀了很多 Atwood，喜歡她的推想小說風格，能讓你想「這真的可能發生在某個社會」）和 Haruki Murakami（喜歡他的魔幻寫實風格）。但她最推薦每個人至少一年讀一次的書是《小王子》（The Little Prince），提醒自己什麼才是真正重要的事。
人生座右銘？工作上：Keep it simple，專注在你真正要做好的那件事。人生上：「在你可以成為任何東西的世界裡，選擇善良。」（In a world where you can be anything, be kind.）她說你永遠不知道別人的生活正在經歷什麼，一個小小的善意舉動可能帶來最大的改變。
她分享了 COVID 期間的故事。那時她一對一會議排得很滿，但奶奶在加拿大的養老院身體不好，因為 COVID 不能去探望。有一天突然接到阿姨的訊息說奶奶今天中午可以 FaceTime，但她有一個很重要的一對一。她硬著頭皮跟下屬說「非常抱歉，這麼臨時，可以改時間嗎？」對方說「當然沒問題。」這對她來說是一個巨大的善意，讓她能跟奶奶在 FaceTime 上打招呼。

▍ 織毛衣的工程主管
Boris Cherny 跟 Lenny 說，在很重要的會議裡，你經常能聽到 Fiona 在背景裡織毛衣的聲音。Fiona 笑著展示了她最近織的上衣。她說她是奶奶在她八歲時教她織毛衣的，每次織的時候都會想到奶奶。
她說織毛衣跟程式設計有很多共通點，兩種針法 knit 和 purl 就像 0 和 1，堆疊（stack）和佇列（queue）的概念都能對應到。「我就像一個 compiler，把指令編譯成一個可執行的成品。」她練到不用看手就能織，所以開會或等待的時候就是她的織毛衣時間，就像有些人用指尖陀螺一樣。她承認自己可能有輕微的毛線囤積症，如果不持續消耗庫存，毛線只會越堆越多。
她的夢想是以奶奶的名字開一家毛線店，創造那種社群感。Lenny 說到時候 Cowork 可以幫她處理所有的發票和帳務。Lenny 也提到他問 Boris 當 AGI 到來不用工作時會做什麼，Boris 說他要做音樂，Fiona 的答案很可能就是開毛線店和織更多衣服
